# Secondary Indexes（二级索引）

> 适用于微服务架构的接口契约、事件、配置、存储等专题文档。
>
> **本项目说明**: 此为个人知识库，非代码项目，因此大部分二级索引不适用。
> 保留此文件作为模板，并记录不适用原因。

---

## 为什么不适用

### 1. Service Topology（服务拓扑）
**不适用原因**:
- 本项目是静态文档库，无服务架构
- 不存在API Gateway、BFF、服务间调用

**替代方案**:
- 本知识库的"内容拓扑"详见 `module-index.md`

---

### 2. API Contract（接口契约）
**不适用原因**:
- 无后端API
- 无OpenAPI/Swagger定义
- 无GraphQL Schema

**替代方案**:
- 文档间的引用关系详见 `functional-inventory.md` 的"跨文档引用关系"部分

---

### 3. Event/Messaging（事件/消息）
**不适用原因**:
- 无事件驱动架构
- 无消息队列（Kafka/RabbitMQ）
- 无Webhook

**替代方案**:
- 知识传递流程详见 `flow-index.md`

---

### 4. Configuration Matrix（配置/环境矩阵）
**不适用原因**:
- 无环境变量配置
- 无多环境部署（dev/staging/prod）
- 无feature flag

**替代方案**:
- 文档格式规范详见 `project-overview.md` 的"文档生产流程"部分

---

### 5. Data Ownership（数据归属）
**不适用原因**:
- 无数据库
- 无数据表归属问题

**替代方案**:
- 领域模型详见 `domain-model.md`

---

## 结论

本项目为**纯文档知识库**，无代码、无服务、无数据库，因此标准二级索引（服务拓扑、接口契约、事件、配置、数据归属）均不适用。

当前的知识组织方式通过：
- `module-index.md`（模块索引）
- `domain-model.md`（领域模型）
- `flow-index.md`（流程索引）
- `functional-inventory.md`（功能清单）

已经覆盖了二级索引的核心目的：**可导航、可追溯、可复用**。

未来如需扩展为交互式平台，再根据实际架构设计二级索引。
